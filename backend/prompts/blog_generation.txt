SEO & READABILITY
• Target keyword density 1–2 %: naturally weave primary keywords into headings and body
• Use short paragraphs (3–5 sentences max); avoid walls of text
• Inline code (`backticks`) for all identifiers, function names, and hyperparameters
• Prefer active voice; avoid filler phrases ("In conclusion, ...", "It is worth noting that …")
• Add anchor-friendly heading slugs — keep ## headings concise and keyword-rich

MARKDOWN FORMATTING
• Only ## and ### headings (NOT # — the page template supplies the H1)
• Blank line before and after every heading, list, table, and code block
• Tables use standard GFM pipe syntax with an alignment row
• No raw HTML — pure markdown only
• Escape any literal curly braces in code as \{{ \}} so they are not mistaken for template variables
